# Derived instances from Yu et al. (2022) — identical SMALLER machine

派生自 `Instances/Yu et al., 2022/`,用于本工作的**同构平行机** 1D-AM 批调度。
相对 `Derived_Yu2022_identical`(取较大机器,导致很多实例无延误而平凡),本集**取较小
机器**以获得区分度。

## 派生规则

- **机器 = 源实例里平台面积 `L×W` 最小的那台**,沿用其 `V, U, S, L, W`。
  (小平台 + ht1 组的高 `U=0.7, S=2` → 批次紧、recoat 慢 → 延误显著。)
- **每个零件的朝向**:orientation-1 若其**底面** `l≤L 且 w≤W` 能塞进就用 ori-1;否则用
  列表里**第一个底面能塞进的朝向**(把细高零件立起来)。体积只用零件体积(忽略支撑)。
- **平台高度 H** = `max(小机原 H, 所有零件所选朝向的最大高度)`,保证每个零件都装得下;
  **L×W(面积容量)保持不变 → 批次紧度完全保留**。已验证每个超标零件都有"底面可行朝向"。
- **交期(方案 A:沿用)**:TestInstances 直接用文件里的交期;LargerInstances(`ht2_1`,
  无交期)用 Yu 的 `GenerateDueDate` 按 4 个 (TF,RDD,seed) 组合生成。
  > 注:Yu 交期按**原异构机** makespan 标定;放到更慢/更紧的小机上等效更紧 → 延误天然变多
  > (这正是本集相对 `_identical` 不再平凡的原因)。名义 TF/RDD 在小机上偏紧,论文需说明一句。
- **M ∈ {2,3,4}** 运行时传入。
