================================================================
 new_oracle 公平时限实验包  (vs MILP/Gurobi)
================================================================

目的:在和 MILP 一样的时限(默认 3600s/算例)下,看 new_oracle 能把
n=16..20 证到什么程度 / gap 压到多少。

----------------------------------------------------------------
怎么用(最省事)
----------------------------------------------------------------
1) 双击  run_exp.bat
   - 若没有 oracle.exe,它会用 g++ 自动编译 oracle.cpp。
   - 没有 g++ 的话,先手动编一次(任选其一),再双击:
       g++ -std=c++17 -O2 -o oracle.exe oracle.cpp
       cl /O2 /EHsc /std:c++17 oracle.cpp /Fe:oracle.exe   (VS 开发者命令行)
   - 想先快速试:  在命令行里跑   run_exp.bat 600     (每算例 600s)

2) 结果写到  results_new_oracle.txt
   每个算例一段,RESULT 行里看这几个值:
     obj      = 找到的最优/最好目标值(总延误)
     proven   = 已证最优;  (TIMEOUT) = 到时限未证完
     lb       = 到时限时的已证全局下界
     gap      = (obj-lb)/obj 的百分比;proven 时为 0.00%
     time_s   = 耗时

----------------------------------------------------------------
算例集  exp_instances\   (共 17 个)
----------------------------------------------------------------
  5..15part, 10part_2/3, 10parts_4, 15part_2-S   —— 仓库原有算例
  16part,17part,18part,19part                    —— 按论文 TF=RDD=0.6、
                                                     平台 20x20 生成(固定种子)
  20part_3-S, 20part_4-S                         —— 仓库原有 20 件算例

格式与 data\ 完全一致(末尾带 DueDate 段),所以**同一批文件也能直接喂给
你的 Gurobi 脚本** experiments\milp\ 做公平对比(把它指向 exp_instances\ 即可)。

----------------------------------------------------------------
时间预估
----------------------------------------------------------------
n<=15 的算例都是秒级;真正吃时间的是 16..20(每个最多 TL 秒)。
默认 TL=3600 时,跑完全部最坏约 6~8 小时(挂着跑即可)。
只想看大算例可临时把小算例移出 exp_instances\,或先用 run_exp.bat 600 试。

----------------------------------------------------------------
说明
----------------------------------------------------------------
- oracle 默认参数:interch 开、LB_pos 开、浅层门控 qfrac=0.5、自适应 BFS&DFS。
  如需调参:oracle.exe <inst> <TL> <N_MAX> <N_MIN> <interch0/1> <pos0/1> <gamma> <qfrac>
- 16..19 为合成算例(论文同分布,非论文原始 GeneratedResults),用于规模演示;
  与 Gurobi 对比时务必两边用同一批 exp_instances\ 文件。
