# 单机 oracle 下界增强:实验结论汇总

目标:把根/浅层下界抬上去,压低 n≥18 的 optimality gap(20part 在 40s 时 gap≈72%,瓶颈在 **bound 侧**)。
下面记录试过的所有下界方向及实测结论。所有"根界 / 最优"百分比在 `data/12–15part` 上测得。

## 现状基线

- 当前生效下界 = `closed(精确) + 开口批次 floor + max(LB_par, LB_pos)`。
- 根界水平:**LB_par ≈ 9%、LB_pos ≈ 14%** of opt。LB_pos 是目前最强的廉价界(它已正确处理:面积→批次数的 setup、划分最小高度、体积累加、交期排序配对)。

## 试过的方向与结论

| 方向 | 根界/opt | 结论 |
|---|---|---|
| **LB_par**(并行/合并取小,逐件) | ~9% | 现有,廉价但松 |
| **LB_pos**(位置界 iron 形式) | ~14% | 现有**最强廉价界**;已接入 |
| 联合收紧 θ_k(体积+线性面积对同一 k-子集) | ≈ LB_pos(+0%) | **无效**:体积项极小(V≈0.03),面积 ceil 已用合法最小,联合几乎不改变 θ_k(13part 节点 0% 变化) |
| **LB_ser**(Zixuan,体积当 1‖ΣT + 精确 DP 排序) | **0–9%** | **比 LB_pos 还弱**,n=14/15 直接塌成 0;只收一次 setup + 一次 h_min,把主导的 setup/高度低估 ~10×;精确 DP 排序用在最小的体积项上,白费。**不移植** |
| **LP/拉格朗日**(时间索引指派 + 机器负载容量,对偶容量、逐件分解、次梯度) | **0–0.3%** | **塌掉**:可线性化的机器负载权重 `w_j=V·vol+(S+U·h_min)/A·area` 太小(总和 ~30,真实 makespan ~100+),容量几乎不绑定 ⇒ 强制完工被低估 ~3× ⇒ 看不到延误 ⇒ 界≈0 |

## 核心诊断:为什么廉价界都抬不动

这个问题的机器时间被 **per-batch 的高度 `U·max h`(尤其高件)和满 setup `S`** 主导,而**体积 `V·vol` 极小**(V≈0.03)。

- 高度是"每批取 max",**不是可加的逐件量**;任何把成本线性化/逐件分摊到 part 的松弛(LP 容量、拉格朗日对偶、LB_ser 的单 setup)都会**丢掉高度这块大头** ⇒ 界塌。
- LB_pos 之所以是廉价界里最强的,正因为它**没有线性化**——它按位置用"面积→批次数×S + 划分最小高度"把这两块大头算进去了。
- 也就是说:**廉价 bound 的天花板就在 LB_pos(~14%)附近**。再想往上,必须用"看得见真实批次(及其真实 max 高度)"的界。

## 唯一还能抬界的方向(代价大)

**列生成 / Branch-and-Price**:列 = 真实批次(带其真实 vol/area/max-h)+ 时间位置;主问题用"每件覆盖一次"的集合划分 LP,定价子问题在时间轴上挑负 reduced-cost 的批次序列。
- 它能看见真实高度/setup ⇒ 界可望接近最优(并行机版 `new_algorithm` 实测根界 ~99%)。
- 但**定价子问题是序列耦合的指数级问题**(`new_algorithm` 已实测每节点过贵),是数周量级的研究型工程;单机情形没有并行机那种"按机器分解"的便宜杠杆。

## 最终建议(诚实)

1. **bound 侧的廉价改进已经做尽**:LB_par/LB_pos/联合/LB_ser/LP-拉格朗日 都试过,LB_pos(~14%)是廉价天花板,其余都 ≤ 它甚至塌成 0。**不要再在廉价界上投入**。
2. 当前 oracle 已经很强:**n≤17 数十秒证到最优**,n=18–20 给强增量解。这对一篇方法论文/基准已经够用。
3. 若研究目标是把"可证最优"推到 n≥20,**唯一路线是 Branch-and-Price**(真实批次定价),属高难度长期方向,且单机定价子问题本身就是难点。
4. 实用折中:n=18–20 用现 oracle 跑长时限(本机 600–3600s)拿强增量解 + 报告 gap(已支持 `gap` 输出),不追求证完。

> 复现:`/tmp` 下的 `lag_probe.py`(LP/拉格朗日根界)、`eval_ser.py`(LB_ser 根界);oracle 的 `rootLB`/`gap` 输出可看任意算例的根界与间隙。
