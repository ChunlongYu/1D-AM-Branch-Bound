# 子问题改写为显式 labeling/DP:可行性与预期提速评估

> 结论先行:**显式 labeling/DP 可行,但只是常数倍提速(约 2–10×),不能跳出指数。**
> 因为子问题不存在干净的多项式 DP 结构(实测见下)。更高性价比的提速在"减少精确子问题
> 求解次数"(Lagrangian 的便宜界 + diving),而非把精确求解器换成 labeling。

## 1. 关键问题:子问题有没有"小状态"的 DP?

labeling/DP 要高效,需要状态**不含完整集合**(像 B&P 对路由/TWCT 那样用 (位置/末点, 资源))。
对本子问题(单机面积约束批调度、总延误),最有希望的小状态假设是:
**"按交期排序 + 只用连续批次、按序加工"即最优** —— 若成立,则 DP 只需对排序序列做"切分",
状态 ~ (位置, 完工时间),近多项式。

## 2. 实测:该假设不成立(决定性)

对 15part 数据的随机子集(大小 4–10,共 280 例),比较
"全枚举有序批次划分的真最优 Φ_full" 与 "EDD 排序连续批次的 Φ_eddcons":

```
匹配 113/280;不符 167 例;最大相对差 203%
```

即面积约束让**非连续批次**(交期相差大但能拼面积的件同批)常常更优,且最优批次顺序 ≠ EDD。
=> **没有连续-EDD 的多项式 DP**;labeling 状态**必须带"已排零件集合"** => 状态数 ~ 2^n,
与现有 B&B 同阶。

## 3. 那么显式 labeling/DP 能带来什么?

现有 `prize_oracle` 加了支配规则后,已是"DFS 形式的 labeling"
(label = (已排集合掩码, C, G),按集合分组支配,每个非支配 label 大致只展开一次)。
改写成**显式前向 labeling**(按 C 分桶、Pareto 支配、拓扑/最优优先扩展)能多拿到的:

- **避免 DFS 的无用深潜**:DFS 在首次到达某 label 时就展开,即便稍后出现更优的同集合 label
  把它支配——之前那次展开的子树就白做了。最优优先(按 C 扩展)先扩展更优 label,
  把劣 label 在展开前就支配掉 => 少做无用功;
- **更快/更强的 dominance**(按 C 分桶);
- 去掉递归开销。

**预期提速:约 2–10×(常数倍),不改变指数复杂度。** 状态本质仍是 (集合, 完工时间)。

## 4. 工程量与风险

- 可行性:**高**。状态、支配、prize-collecting / 强制基集都已清晰,前向 labeling 直接可写。
- 工程量:中(重写求解核 + 桶结构 + 对拍验证仍 24/24)。
- 风险:低(纯实现层;正确性可对拍)。但**回报有限**(常数倍)。

## 5. 建议的优先级(性价比排序)

1. **(最高)Lagrangian 的"便宜后精确"**:Lagrangian 只需子问题的**合法下界**。多数次梯度步用
   prize oracle 里已有的 **O(n) 带奖励并行界**(不跑搜索),仅在界接近 UB 时才精确求解。
   这直接砍掉"每节点 × 迭代 × M 次精确求解"里的绝大多数——**与求解器快慢无关,收益更大**。
2. **diving 收紧上界**(复用基础 B&B 的均衡 diving + 守门叶子):上界紧 => 节点少、子问题调用少。
3. **(其次)显式 labeling/DP**:在 1、2 之后若仍需精确子问题更快,再做,稳拿 2–10×。
4. 跨迭代/跨节点缓存(注意 u 变化,缓存价值有限)。

## 6. 与"走向 B&P"的关系

B&P 对 **TWCT** 的 labeling 能近多项式(Yu et al. 因此上百件),其前提正是 TWCT 允许小状态;
本问题(**总延误 + 面积批次**)实测无此结构 => **tardiness 版的 labeling/pricing 本就偏难**,
这也再次说明完整 B&P 走 tardiness 是高难度的长期路线(见 `BPC_Analysis_Yu_et_al.md`、
`Pricing_Labeling_Connection.md`)。
