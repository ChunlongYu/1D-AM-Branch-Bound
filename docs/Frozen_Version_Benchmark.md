# 固化版本 + 对比实验报告

本文把当前 oracle / 并行 B&B **固化为一个基准版本**,并给出三组对比实验:
单机正确性回归、并行机 B&B 消融(M=2/3/4)、规模扩展(n=16–20)。
所有运行用 `g++ -O2` 在统一环境下完成;每条数据可由文末命令复现。

---

## 0. 固化的版本(冻结配置)

### 单机 oracle(`new_oracle/oracle.cpp` → `oracle`)

| 项 | 默认值 |
|---|---|
| 分支 | Type I / Type II(`j>oMax` 递增对称破除) |
| 下界 | `closed(精确) + 开口批次 floor + max(LB_par, LB_pos)` |
| `interch` | 1(相邻批次交换支配,开) |
| `pos` | 1(LB_pos 开) |
| `qfrac` | 0.5(LB_pos 浅层门控:`|R| ≥ 0.5n` 才算) |
| `gamma` | 0 |
| `N_MAX / N_MIN` | 200000 / 50000(自适应 BFS&DFS 切换阈值) |
| `FRONT_CAP` | 25,000,000(支配前沿键数内存上限) |

构建:`cd new_oracle && g++ -std=c++17 -O2 -o oracle oracle.cpp`
运行:`./oracle <inst> <TL秒> [N_MAX] [N_MIN] [interch] [pos] [gamma] [qfrac] [FRONT_CAP]`

### 并行机 B&B(`src/ParallelBranchBound.cpp` → `pbb`)

| 项 | 默认值 |
|---|---|
| 方案 | 方案 A:零件→机器指派分支,叶子用单机 Φ 精确求值 |
| Φ | `new_oracle/phi_oracle.h::solvePhi`(即上面的强 oracle,~5× 旧版) |
| diving | **depth-first**(深度优先下潜:最深优先,平手取 min ℓ;`DIVE=load` 退回旧 min-ℓ 规则) |
| `LS` | 1(move 邻域局部搜索初始上界,**开**) |
| `WARM` | 0(显式暖缓存,关;LS 开时基本冗余) |
| `FREEPAR` | 0(并行位置自由零件界,关;已证被 singleTard 压制) |
| `HEAVY` / `HEAVYMARGIN` | 1 / **2(标定值)**。机器件数 ≥ K=⌈n/M⌉+`HEAVYMARGIN` 时,用小预算(n² 节点)的单机 oracle LB 顶替 lbPos≈0,在内部节点剪掉极不平衡的垃圾子树。`HEAVY=0` 关。margin 由 DOE 主效应取 2(效应在噪声量级) |
| `MOVEBUDGET` | **0.3(标定值)**(**构造/move 阶段预算**,0.3·TL)。构造阶段每次精确 Φ 受 `min(剩余, 0.3·TL)` 限时,防止单个大子集吞掉整段时间致主循环 0 节点。非 proven 限时结果作 UB 但**不入缓存**(保 lbMem)。`MOVEBUDGET=0` 关。安全阀,DOE 中效应在噪声量级 |
| `N_max / N_min` | 200000 / 50000 |
| `SCORE`(分支打分) | **spread(标定值)**;强分支按 child-LB 的 spread=max−min 打分。DOE:spread/sum 并列优,**min 最差**(主导因素之一) |
| `CAND`(`strong_branch_candidates`) | **8(标定值)**;强分支候选件数。DOE 最强主效应:k=8 最优,**k=4 明显最差** |

构建:`cd src && g++ -std=c++17 -O2 -o pbb main.cpp ParallelBranchBound.cpp BranchBound.cpp InstanceData.cpp`
运行:`./pbb <inst> <M> <TL秒>`

## 2. 并行机 B&B:diving 规则修正 + 消融(M=2 / 3 / 4)

本轮把搜索内的 "diving" 规则从旧的 **min-ℓ(均衡优先)** 改成 **depth-first(深度优先下潜:最深节点优先,平手取最均衡)**。两者最优值完全一致,只比效率与内存。`peak` = 活动节点表的峰值规模(内存代理指标)。

### 2.1 Diving 规则对比(头条)

| 实例 | M | min-ℓ 节点 | min-ℓ peak | **depth 节点** | **depth peak** |
|---|---|---|---|---|---|
| 13part | 2 | 5,945 | 1,683 | 5,065 | **465** |
| 13part | 3 | 97,990 | 30,469 | **5,119** | **366** |
| 13part | 4 | 25,277 | 6,926 | **1,293** | **23** |
| 14part | 2 | 9,781 | 1,662 | 6,397 | **321** |
| 14part | 3 | 15,378 | 4,777 | 6,399 | **398** |
| 15part | 2 | 22,305 | 6,269 | 9,865 | **1,108** |
| 15part | 3 | 93,975 | 14,905 | **3,399** | **23** |
| 16part | 3 | 184,275 † | 122,602 | 52,173 | 4,413 |
| 18part | 3 | 65,022 † | 129,940 | 29,255 | **33** |

> † = 在 30–35s 时限内**未证完**;depth 在这两例分别 4.2s / 1.08s **证到最优**。

读数:

- **峰值前沿坍塌 3–4 个数量级**(15part M=3:14,905 → 23;18part M=3:129,940 → 33)。这才是 `dfsMode` 内存阀门本该有的行为——旧 min-ℓ 是按负载层级广度扫,前沿只涨不收,正是 n=19/20 OOM 的根因。
- **节点数在 M≥3 大幅下降**(15part M=3:94k → 3.4k;13part M=3:98k → 5.1k)。
- **大算例上 min-ℓ 证不完而 depth 几秒证完**;16part M=3 里 min-ℓ 跑满 30s 增量解还卡在 25.27,depth 很快潜到真最优 20.68。

### 2.2 局部搜索(LS)边际效应(depth 默认下)

换上 depth 下潜后,LS 的边际收益明显变小(下潜本身就快速拿到均衡好叶子)。Default(LS 开) vs LS off,生成节点:

| 实例 | M | Default(LS 开) | LS off |
|---|---|---|---|
| 13part | 2 | 5,065 | 5,215 |
| 13part | 3 | 5,119 | 6,127 |
| 13part | 4 | 1,293 | 4,457 |
| 14part | 2 | 6,397 | 5,877 |
| 14part | 3 | 6,399 | 2,445 |
| 15part | 2 | 9,865 | 9,881 |
| 15part | 3 | 3,399 | 8,910 |
| 15part | 4 | 0 | 1,546 |

读数:LS 现在是**小幅、依实例**——15part M=3/M=4 仍有用,14part M=3 反而略增。下潜规则修对后,LS 之前干的"快速收紧增量解"大部分被下潜本身吃掉,两者部分冗余。默认仍开(便宜、永不抬高最优)。

### 2.3 FREEPAR 仍是 no-op

新默认下复测:15part M=2/M=3 开关 FREEPAR 节点数逐位相同(被 singleTard 结构性压制,与节点顺序无关)。默认关。

## 3. 规模扩展(n=16–20,单机 oracle)

合成 16–19 件(论文 TF=RDD=0.6、20×20 平台、固定种子)+ 仓库 20 件算例。
本表时限为**沙箱短时限**(32–35s,仅作规模演示);你本机用 600–3600s 可证更多
(例:你此前 3600s run 里 `20part_3-S` 可在 ~218s 证完)。gap = (obj−lb)/obj。

| 实例 | n | obj(最好解) | 状态 | lb(已证下界) | gap | 时间 |
|---|---|---|---|---|---|---|
| 16part | 16 | 304.217 | **可证最优** | 304.217 | 0.00% | 12.4s |
| 17part | 17 | 206.31 | **可证最优** | 206.31 | 0.00% | 15.8s |
| 18part | 18 | 232.734 | 增量解(超时) | 133.637 | 42.6% | 35s |
| 19part | 19 | 352.096 | 增量解(超时) | 124.476 | 64.7% | 35s |
| 20part_3-S | 20 | 224.652 | 增量解(超时) | 62.18 | 72.3% | 32s |
| 20part_4-S | 20 | 240.99 | 增量解(超时) | 67.98 | 71.8% | 32s |

### 读数

- **n≤17:十几秒内证到最优**(16part 12.4s、17part 15.8s)。
- **n=18–20:短时限下给强上界,但 gap 大**——瓶颈在**下界侧**(根/浅层界~14% of opt,廉价界天花板),不是上界。增大时限主要靠枚举把 gap 磨下来,不是界变紧。
- 这与 `Bound_Strengthening_Findings.md` 的结论闭合:**廉价合法界已做尽,n≥18 想"证完"只有 Branch-and-Price 这条重型路。**

## 4. 一页结论

| 维度 | 状态 |
|---|---|
| 单机正确性 | ✅ 全量 11 算例与基准一致、全部可证最优 |
| 单机规模 | n≤17 数十秒证完;n=18–20 给强增量解(界受限) |
| 并行 diving | **depth-first 取代 min-ℓ**:峰值前沿降 3–4 数量级,M≥3 节点降 1–2 数量级,大算例从证不完→几秒证完 |
| 并行 M=2 | LB-limited,初始上界已近最优;diving 改进主要体现在内存/节点常数 |
| 并行 M≥3 | depth 下潜主导收益;LS 退为小幅依实例的廉价精修 |
| FREEPAR | 零效果(已证死路),默认关 |
| 暖缓存 | 与 LS 部分冗余;depth 默认下作用进一步减小 |

**固化结论**:当前 DEFAULT 配置在正确性与效率上均为各规模的稳态最优,可作为后续工作的基准版本。
