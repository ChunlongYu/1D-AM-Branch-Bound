# Type I/II 单机 oracle 的支配规则推导

面向问题:单机一维(面积约束)AM 批调度,目标 $\min\sum_j T_j$。分支用 Type I(开新批次)/ Type II(并入当前开口批次,下标递增)。

> **先回应大问题:用新分支 + 支配规则会更好吗?**
> 会——但有两个前提。① 下界必须先改成**合法**的(见 `给Zixuan_下界问题说明.md`,否则无论怎么加支配都还是会给错最优);② 支配的"键"必须包含**开口批次**、并且比较的是**已封口批次的延误**而不是总延误——上一版支配规则正是因为漏了这两点而失效。把这两点做对,支配能让节点数显著下降(尤其在 best-first 下),通常也能换来 wall-time 改善,但每个节点要多付一次"建键 + 帕累托比较"的开销,最终是否更快要实测(参见论文里 $BB_{ser}$ 节点更少但时间更长的现象)。

---

## 1. 记号与节点状态

零件 $j$:体积 $v_j$、高度 $h_j$、投影面积 $a_j=l_jw_j$、交期 $d_j$。平台面积 $A=LW$。批次加工时间
$$P(B)=S+V\!\!\sum_{i\in B}v_i+U\max_{i\in B}h_i .$$
批次串行,同批零件完工时间相同。

一个节点是"若干已封口批次 $B_1,\dots,B_{r-1}$ + 一个**开口批次** $B_r$"。只有最后一个批次是开口的(Type II 只能往它里面加;一旦 Type I 开新批次,前一个就永久封口)。

对节点 $x$ 定义其**状态签名**:

| 量 | 含义 |
|---|---|
| $\mathcal S_x$ | 已排零件集合($\mathcal U_x=J\setminus\mathcal S_x$ 为剩余) |
| $B_x$ | 开口批次的**内容**;由此得 $m_x=\max_{i\in B_x}i$、面积 $\alpha_x$、体积 $\nu_x$、高度 $H_x$ |
| $t^-_x$ | 开口批次的**起始时间** = 它之前所有(已封口)批次的完工时间 |
| $TT^{\mathrm{cl}}_x$ | **已封口批次**($B_1..B_{r-1}$)产生的总延误(**不含开口批次**) |

派生量:开口批次当前完工 $C_x=t^-_x+S+V\nu_x+UH_x$;当前总延误 $TT_x=TT^{\mathrm{cl}}_x+\sum_{i\in B_x}\max(0,C_x-d_i)$。

**为什么必须这样选状态。** 从 $x$ 出发的任何延拓都是:先往开口批次 $B_x$ 里追加某个 $E\subseteq\mathcal U_x$(下标都 $>m_x$、面积 $\alpha_x+a(E)\le A$),把它最终封口,再把剩下的 $\mathcal U_x\setminus E$ 排成新批次。于是从 $x$ 能达到的最优值
$$Z(x)=TT^{\mathrm{cl}}_x+\min_{E}\Big[\sum_{i\in B_x\cup E}\max\!\big(0,C_x(E)-d_i\big)+\Phi\big(\mathcal U_x\setminus E,\;C_x(E)\big)\Big],$$
其中 $C_x(E)=t^-_x+S+V(\nu_x+v(E))+U\max(H_x,\max_{i\in E}h_i)$,$\Phi(R,t)$ 是"把零件集 $R$ 从 $t$ 时刻起排成批次序列的最小总延误"(关于 $t$ 单调不减)。

关键:$Z(x)$ 只依赖 $\big(\mathcal S_x,\,B_x,\,t^-_x,\,TT^{\mathrm{cl}}_x\big)$。**开口批次 $B_x$ 在里面**——因为它决定了还能并入哪些零件、以及这些零件(连同 $B_x$ 里已有零件)的完工。上一版支配只用 $\mathcal S$ 和总延误 $TT$,丢了 $B_x$,也把"会随 Type II 变大的开口批次延误"算进了比较量,所以不成立。

## 2. 主支配引理(所有规则的基础)

> **引理.** 设两节点 $u,v$ 满足
> $$\mathcal S_u=\mathcal S_v,\qquad B_u=B_v\ (\text{开口批次内容完全相同}),$$
> 且
> $$t^-_u\le t^-_v\quad\text{且}\quad TT^{\mathrm{cl}}_u\le TT^{\mathrm{cl}}_v,$$
> 则 $Z(u)\le Z(v)$,即 $u$ 支配 $v$(任一不等式严格则严格支配,可剪掉 $v$)。

**证明.** $B_u=B_v=:B$ 且 $\mathcal U_u=\mathcal U_v$,故两者可行的追加集合 $E$ 完全相同。对任意 $E$:
$$C_u(E)-C_v(E)=t^-_u-t^-_v\le 0\ \Rightarrow\ C_u(E)\le C_v(E).$$
于是(i)开口批次 $B\cup E$ 中每个零件在 $u$ 的完工 $\le$ 在 $v$ 的完工,延误项逐个 $\le$;(ii)尾部从更早的 $C_u(E)$ 开始,$\Phi$ 单调 $\Rightarrow \Phi(\cdot,C_u(E))\le\Phi(\cdot,C_v(E))$。再加 $TT^{\mathrm{cl}}_u\le TT^{\mathrm{cl}}_v$,得相同 $E$ 下 $u$ 的括号值 $\le v$ 的;对 $E$ 取 min 不改变 $\le$。$\square$

**两个必须强调的修正(相对上一版):**
1. 键里要含**开口批次内容** $B$(不能只用 $\mathcal S$)。
2. 比较的延误是**已封口部分** $TT^{\mathrm{cl}}$(以及起始时间 $t^-$),**不是**含开口批次的总延误 $TT$。原因:开口批次的延误会随后续 Type II 增大,把它放进比较量会得出错误支配。等价的可观测形式是比较 $(C_x,\,TT^{\mathrm{cl}}_x)$,因为 $B$ 相同则 $C$ 与 $t^-$ 一一对应。

实现上:对每个键 $(\mathcal S,B)$ 维护一个关于 $(t^-,TT^{\mathrm{cl}})$ 的**帕累托前沿**,新节点若被前沿中某点支配则剪,否则入前沿并删去被它支配的旧点。

## 3. 三类规则

按"开口批次是单件还是多件"给两节点分类(对刚生成的节点,这恰好对应最后一步是 Type I 还是 Type II):**type-1 = 开口批次为单件**,**type-2 = 开口批次为多件**。

### 类 1:type-1 ↔ type-1(两个开口批次都是单件)

同一 $\mathcal S$、开口批次都恰为某单件 $\{p\}$ ⇒ 开口批次内容相同(都是 $\{p\}$),直接套**主引理**。差别只在前缀(把 $\mathcal S\setminus\{p\}$ 排成不同的封口批次),即不同的 $(t^-,TT^{\mathrm{cl}})$。

> **规则 1.** 在所有"已排集合 $=\mathcal S$、开口批次 $=\{p\}$"的节点里,
> $$\big(t^-_u\le t^-_v\big)\ \wedge\ \big(TT^{\mathrm{cl}}_u\le TT^{\mathrm{cl}}_v\big)\ \Rightarrow\ u\prec v.$$

**会触发的例子**:$\mathcal S=\{1,2,3\}$,开口批次 $\{3\}$。前缀把 $\{1,2\}$ 排成 $\{1\}\{2\}$ 或(若面积允许)$\{1,2\}$,这两种给出不同的 $(t^-,TT^{\mathrm{cl}})$,取被支配者剪掉。

### 类 2:type-2 ↔ type-2(两个开口批次都是多件)

同样是主引理的特例:同一 $\mathcal S$、开口批次内容同为多件集合 $B$。

> **规则 2.** 在所有"已排集合 $=\mathcal S$、开口批次内容 $=B$"的节点里,
> $$\big(t^-_u\le t^-_v\big)\ \wedge\ \big(TT^{\mathrm{cl}}_u\le TT^{\mathrm{cl}}_v\big)\ \Rightarrow\ u\prec v.$$

注意"开口批次内容相同"已隐含 $m,\alpha,\nu,H$ 都相同,所以后续可追加集合、追加边际成本都一致,引理直接适用。

> ⚠️ **不要做的事**:不要去比较"开口批次内容**不同**但 $\mathcal S$ 相同"的两个 type-2 节点(例如开口批次 $\{2,4\}$ vs $\{3,4\}$),也不要去比较"加入零件 $q$"与"加入零件 $q'$"这种**已排集合不同**的两个 Type II 子节点。它们的可行后续(受递增下标 + 面积约束)不同,$\Phi$ 的总延误对总加工时间也非线性,**不存在**这种情形下的普适支配(这正是上一版出错的同类陷阱)。

### 类 3:type-1 ↔ type-2(封口 vs 合并 —— 唯一"有条件"的一类)

最有用的代表是**同一父节点、加入同一零件 $p$** 的两个兄弟(此时它们的 $\mathcal S$ 相同):

设父节点开口批次 $B_0$($t^-$、$\nu_0$、$H_0$、$\alpha_0$、$m_0$),其当前完工 $C=t^-+S+V\nu_0+UH_0$。

- **合并** $u_p$(Type II,需 $p>m_0$ 且 $\alpha_0+a_p\le A$):开口批次 $B_0\cup\{p\}$,$t^-_{u}=t^-$,$TT^{\mathrm{cl}}_{u}=TT^{\mathrm{cl}}_{\text{父}}$。
- **封口** $w_p$(Type I):$B_0$ 在 $C$ 封口,开口批次 $\{p\}$,$t^-_{w}=C$,$TT^{\mathrm{cl}}_{w}=TT^{\mathrm{cl}}_{\text{父}}+\sum_{i\in B_0}\max(0,C-d_i)$。

两者开口批次内容不同 ⇒ 主引理不适用,要直接比 $Z$。对**任意相同追加集 $E$**(对 $u_p$ 可行,即 $\alpha_0+a_p+a(E)\le A$),逐项相减得
$$
\underbrace{\text{封口}-\text{合并}}_{=\,Z\text{ 的同-}E\text{项之差}}
=\underbrace{\sum_{i\in B_0}\!\big[\max(0,C-d_i)-\max(0,C_u(E)-d_i)\big]}_{\le 0:\ B_0\text{ 在封口下更早,获益}}
+\underbrace{\sum_{i\in\{p\}\cup E}\!\big[\cdots\big]+\big[\Phi(\cdot,C_w(E))-\Phi(\cdot,C_u(E))\big]}_{\ge 0:\ p\cup E\cup\text{尾部在封口下晚 }\Delta\ge S}
$$
其中 $C_w(E)-C_u(E)=S+U\min\!\big(H_0,\max(h_p,h_E)\big)\ge S$。直观:**封口为 $B_0$ 的零件买来"早完工",代价是给 $p$ 及其后所有零件多付一次 setup $S$。**

由此得两条可判定的充分条件:

> **规则 3a(优先合并 / 剪掉封口).** 若 $B_0$ 的零件在合并后也都**不会延误**——即对所有 $i\in B_0$ 有 $d_i\ge \overline C_u$,其中 $\overline C_u$ 是 $u_p$ 开口批次完工的一个上界
> $$\overline C_u=t^-+S+V\big(\nu_0+\!\!\sum_{j\in\mathcal U,\,j>m_0,\ \text{面积可纳}}\!\! v_j\big)+U\,\textstyle\max_{i\in B_0\cup\{j:\dots\}}h_i,$$
> 则上式第一项 $=0$,故"封口 $-$ 合并 $\ge 0$",$u_p\prec w_p$,**封口子节点可剪**。
>
> 证明:第一项为 $0$ 时,差 $\ge 0$ 对每个可行 $E$ 成立;对封口的最优 $E$,若它对合并也面积可行则直接得 $Z(u_p)\le Z(w_p)$;若不可行(更满),说明封口用到了合并没有的额外面积,但那只会让封口的 $p\cup E$ 更挤更晚,不改变结论方向。$\square$

> **规则 3b(优先封口 / 剪掉合并,较少见).** 仅当"为 $p$ 及尾部多付的 $\Delta$"不产生任何额外延误、而 $B_0$ 又确有零件靠早完工避免延误时,封口才占优。可判定的安全特例:$B_0$ 中存在 $d_i\in(C,\;C_u^{\min}]$ 的零件(合并必使其延误),且 $\{p\}\cup\mathcal U$ 全部在封口后的更晚时刻仍不延误(例如它们交期都很大)。否则**默认保留合并、不要用封口去剪合并**。

实务建议:类 3 不要写成无条件规则;按规则 3a 去剪"被合并支配的封口"即可拿到大部分收益,3b 作为保守补充。

## 4. 实现要点

1. **键** = `(scheduled_set_bitmask, open_batch_bitmask)`。`open_batch_bitmask` 不可省。
2. 每个键维护 $(t^-,\,TT^{\mathrm{cl}})$ 的**帕累托前沿**(二维支配)。注意存的是**封口延误**,需要在节点里单独维护 $TT^{\mathrm{cl}}$(封口一个批次时把该批次延误累加进去),不要用总延误。
3. 比较顺序:生成子节点 → 算 $C,TT^{\mathrm{cl}}$ → 先查类 1/2 帕累托前沿(便宜)→ 再对"同父同零件"的封口/合并对用规则 3a。
4. **正确性自检**:打开支配后,在 `data/` 全部算例上与 `src/` oracle 逐一核对 $TT$ 必须完全一致(尤其 10part_2、10part_3 这两个上一版翻车的算例)。任何不一致都说明某条支配仍不合法。

---

## 5. 一句话结论

- **类 1、类 2** 是主引理的干净特例(同集合 + **同开口批次内容**,比较 $(t^-,TT^{\mathrm{cl}})$),无条件成立、可直接实现。
- **类 3**(封口 vs 合并)本质是"早完工 vs 省一次 setup"的权衡,**没有无条件支配**;规则 3a 给出可判定的充分条件(开口批次零件不会因合并而延误 ⇒ 剪封口),足以拿到主要收益。
- 切忌跨"不同开口批次内容 / 不同已排集合"硬比(类 2 警示框、以及加入 $q$ vs $q'$ 之类),那是上一版同型错误的来源。
- 这些规则只有在**下界已合法**时才有意义;两件事一起做对,新分支 oracle 才可能在中小规模上稳定优于基线。
