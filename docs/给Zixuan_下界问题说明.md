# 关于单机 B&B 下界的一个问题（给 Zixuan）

Zixuan 你好,我把你的单机 oracle(`references/Zixuan_codes/`)和原来的 `src/` 版本做了头对头测试,发现**并行下界 $LB_{par}$ 不是一个合法下界,会剪掉最优解**。下面把现象、根因、反例、验证和修法都列出来,方便你定位。结论是:**分支(Type I/II)本身没问题,问题出在下界。**

## 2. 根因:$LB_{par}$ 漏算了 Type II(并入开口批次)

你论文(NodeGenerationNew)里的并行界,和代码 `compute_unassigned_lower_bound` 完全一致:

$$LB_{par}=TT_{partial}+\sum_{j\in\mathcal{R}}\max\{0,\;C_{now}+S+Vv_j+Uh_j-d_j\}$$

它对每个未排零件 $j$,都假设它**在 $C_{now}$ 处新开一个批次**,完工时间记为 $C_{now}+S+Vv_j+Uh_j$。

问题在于:你的分支里有 **Type II**——未排零件可以**并入当前那个还没封口的批次** $b$。这种情况下零件 $j$:

- **不付新的 setup $S$**(批次 $b$ 的 setup 早就含在 $C_{now}$ 里了);
- 高度项只多付 $U\max(0,\,h_j-H_b)$,而不是 $Uh_j$。

并入开口批次后,$j$ 的真实完工时间是

$$c_j^{\text{TypeII}} = C_{now} + Vv_j + U\max(0,\,h_j-H_b),$$

它**比公式里的 $C_{now}+S+Vv_j+Uh_j$ 小**(至少小 $S+U\min(h_j,H_b)$)。

所以 $LB_{par}$ **高估**了这些零件的完工时间,进而可能让 $LB_{par}$ **大于子树的真最优**。一旦 $LB_{par}\ge UB$,含最优解的节点就被错误剪掉了。论文里"parts 并行 vs 串行所以乐观"的说法只覆盖了"并行/串行"这一面,没覆盖"并入开口批次更便宜"这一面,所以那个乐观性论证是不完整的。

## 3. 一个最小反例(2 个零件就能复现)

机器参数 $S=2,\,V=0.03,\,U=0.7$。两个零件:

- A:$v_A=10,\;h_A=30,\;d_A=1000$(due date 很大,永不延误)
- B:$v_B=10,\;h_B=2,\;d_B=25$

考虑节点 N:开口批次 = {A},$C_{now}=S+Vv_A+Uh_A=23.3$,$TT_{partial}=0$,$\mathcal{R}=\{B\}$。

- **你的下界**:$LB_{par}(N)=\max(0,\;23.3+2+0.3+1.4-25)=\boxed{2.0}$
- **真子树最优**:走 Type II 合成批次 {A,B},完工 $=S+V(v_A+v_B)+U\max(h_A,h_B)=2+0.6+21=23.6\le 25$,B 不延误,A 也不延误 ⇒ $TT=\boxed{0}$

$LB_{par}(N)=2.0 > 0=$ 真最优。当 $UB$ 略大于 0 时,这个本来能拿到零延误的节点会被剪掉。这就是 bug 的本质。

## 4. 三重验证(确认 81.7384 才是 10part_2 的真最优)

1. **重算原 oracle 的最优排程**:B0={0,5,6}, B1={1,3,4,8,9}, B2={2,7},独立按公式重算,面积可行,TT=81.7384。
2. **独立随机搜索**(40 万次):最好值也是 81.7384,从未更低,且大量解低于你的 98.5656。
3. **用 Python 精确复刻你的 Type I/II 规则**枚举整棵树:树内真最优 = 81.7384 ⇒ 分支树里**确实包含**最优,是被下界剪掉的。
4. **把下界改松**(每个未排零件只计 $Vv_j$,去掉 $S$ 和 $Uh_j$)后,你的代码在 10part_2/10part_3 上**恢复正确**(81.7384 / 136.74),但节点暴涨、变慢——印证了"原界因高估而误剪"。

## 5. 论文里需要一起改的地方

- **$LB_{par}$**:公式和"乐观性"论证都要修(见上,要考虑 Type II)。
- **$LB_{ser}$**:证明里"$C_j^{real}$ 至少含一个 setup 和一个高度项"对并入开口批次的零件**不成立**,整段推导默认"剩余零件都另开新批次",同样忽略了 Type II,所以在"开口批次非空"的节点上也不是合法下界。
- **支配规则**:以 $(\mathcal{S}_u,C_u,TT_u)$ 为键,前提是"已排集合相同 ⇒ 后续连续相同"。但 Type II 下,已排集合相同、开口批次构成不同的两个节点,其 Type II 后续并不相同,所以这个前提也不成立。

> 注:代码里 $LB_{ser}$(`compute_unassigned_lower_bound2`)、混合界的 delta 触发、支配规则**目前都被注释掉了**,实际只跑 $LB_{par}$。所以现在体现出来的就是 $LB_{par}$ 这一条的错。

> 另外提醒:论文 Table II 用的是另一套 TF/RDD 生成的算例,可能恰好没触发这个 bug(所以全显示 Opt.、和 Gurobi 一致)。但这是**静默错误**——结果看起来对,不代表方法正确。建议把所有算例的 TT 和 Gurobi/原 oracle 全量交叉核对一遍。

## 6. 修法建议

核心原则:**下界里对"可以并入当前开口批次"的零件,不能强加新的 setup 和满高度。**

一个仍然合法、且比"只算 $Vv_j$"更紧的做法:对每个未排零件 $j$,它的完工时间下界取两种延拓里更小的那个增量——

$$\underline{c_j} = C_{now} + \min\Big\{\underbrace{Vv_j+U\max(0,h_j-H_b)}_{\text{并入开口批次}},\;\underbrace{S+Vv_j+Uh_j}_{\text{另开新批次}}\Big\}$$

(根节点没有开口批次,只有后一项;$H_b$ 为当前开口批次的高度。)然后照旧 $\sum_j\max(0,\underline{c_j}-d_j)$。串行界 $LB_{ser}$ 要按同样思路重做相对偏移。

如果只是先要一个**保证正确**的版本跑实验,也可以直接退用 `src/` 里已验证合法的 `compute_total_lower_bound`,等新界证明清楚再换上。
